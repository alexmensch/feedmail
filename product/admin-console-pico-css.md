## Feature: Admin Console Pico CSS

Adds visual styling to the entire feedmail HTML interface — admin console pages, auth pages, and all public-facing HTML pages (subscription confirmation, verification, unsubscribe, error) — using Pico CSS as the design foundation. The functional console (admin-console-functional) provides plain HTML forms and pages with no visual design. Pico CSS is a semantic-first minimal CSS framework that styles standard HTML elements with professional defaults, built-in dark mode, and fluid responsive design, requiring no build step. All HTML-rendering templates adopt Pico's semantic markup conventions and default styling, with a minimal override stylesheet for feedmail-specific components. The result is a polished, responsive experience across every user-facing and admin-facing page that establishes the baseline visual trust needed for people evaluating feedmail as a serious tool.

**Implementation note:** This feature supersedes the admin-console-styled feature (guid `D04F43C0-AF8F-4CDA-B9ED-4E9C1D3ACA1B`, PR #56). Implementation must begin from the codebase state _before_ PR #56 was merged — that PR's CUBE CSS styling, Every Layout primitives, template markup changes, and static assets (`styles.css` content, layout partials) are fully replaced by this feature. Any non-styling changes from subsequent commits should be preserved.

## Out of scope

- **Functional changes** — No new pages, routes, API changes, or behavioral modifications. The admin console does exactly what it does today.
- **Custom theming or branding** — No feedmail logo, custom color palette, or brand identity. Pico's defaults are the look.
- **Email templates** — `newsletter.hbs`, `newsletter.txt.hbs`, `verification-email.hbs`, `partials/email-footer.hbs`, and `admin-magic-link.hbs` are HTML emails with inline styles, not part of the admin console CSS. These are untouched.
- **JavaScript refactoring** — WebAuthn inline JS, the HTMX library itself, and inline scripts are not being refactored. Only `hx-*` attributes change if container elements change.
- **Pico customization or theming** — No custom Pico theme, no overriding Pico's color palette or typography scale. The default Pico look is accepted as-is.
- **Build pipeline** — No CSS preprocessor, bundler, or build step is introduced. Pico is committed as a static file, same as `htmx.min.js`.
