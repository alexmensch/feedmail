## Feature: Setup Simplification

Reduces the interactive setup script to infrastructure provisioning only — D1 creation, config file generation, credential seeding, and worker deployment — now that the admin console's first-time setup flow handles channel and feed creation. Eliminates Wrangler secrets in favour of D1-stored credentials and deploys both the API and admin workers.

## Out of scope

- **Migration of existing deployers** — this feature applies to new installations only; migrating Wrangler secrets to D1 for existing deployments is a separate concern
- **Changes to install.sh** — the bootstrap script (prerequisite checks, clone, pnpm install, hand off to setup.sh) is unchanged
- **Admin API key rotation or regeneration** — deferred to Admin Console Enhancements feature
- **Resend API key validation during setup** — the script does not verify the key against the Resend API; invalid keys will produce clear errors at email send time
- **Admin email format enforcement beyond basic check** — the setup script performs a basic `@` presence check; full email validation is not in scope
- **wrangler.admin.toml generation** — the admin worker's wrangler config file is assumed to already exist as part of the admin-auth-magic-link feature; this script uses it but does not create the template
