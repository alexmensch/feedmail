#### Feature: Trailing slash normalization

Requests to both the API and Admin workers with trailing slashes (e.g. `/admin/login/`, `/api/subscribe/`) are not matched by route handlers that use exact string comparison, resulting in empty 404 responses or — in the case of bare `/admin` — no worker matching at all and a Cloudflare timeout. This feature normalizes URL paths early in both workers so that trailing slashes are handled consistently.

#### Out of scope

- **API Worker route pattern change:** The API Worker keeps its existing `YOUR_DOMAIN/api/*` route pattern. Bare `/api` is not a meaningful endpoint and does not need to be caught.
- **Redirect behaviour for API Worker:** The API Worker silently strips trailing slashes. No 301/308 redirects — API clients do not benefit from URL bar updates.
- **Canonical URL enforcement beyond trailing slashes:** Other URL normalization (e.g. lowercasing paths, removing double slashes mid-path) is not in scope.
- **Migration script for `wrangler.admin.prod.toml`:** The route pattern change applies to the template `wrangler.admin.toml`. Existing deployers will need to manually update their `wrangler.admin.prod.toml` — a migration script is deferred.

## Technical Specification

### Summary

Requests to the feedmail API and Admin workers with trailing slashes (e.g., `/admin/login/`, `/api/subscribe/`) fail to match route handlers that use exact string comparison, resulting in unexpected 404 responses or Cloudflare timeouts. Additionally, bare `/admin` (without trailing slash) does not match the Admin Worker's Cloudflare route pattern `YOUR_DOMAIN/admin/*`, causing the request to fall through to the origin. This feature normalizes URL paths early in both workers so that trailing-slash variants are handled consistently, and broadens the Admin Worker's route pattern to catch bare `/admin`.

### Implementation Notes

- **Same regex in both workers:** `url.pathname = url.pathname.replace(/\/+$/, "")` handles single and multiple trailing slashes
- **No shared utility:** The logic is 4-6 lines per worker with different behavior (redirect vs silent strip), so inlining is preferred over abstraction
- **Normalization placement:** Before `try` block in both workers, ensuring it runs before rate limiting, session checks, and route matching
- **No changes needed to:** `ROUTE_METHODS`, `PUBLIC_ROUTES`, `getEndpointName()`, `setup.sh`, or existing tests

### Files Changed

1. `wrangler.admin.toml` — route pattern
2. `src/admin/worker.js` — trailing-slash redirect/strip + dashboard route simplification
3. `src/api/worker.js` — trailing-slash strip
4. `package.json` — version bump
