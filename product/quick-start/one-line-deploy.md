## Feature: One-Line Deploy

Replaces the current setup script with a hardened, idempotent shell script that provisions all infrastructure (D1 database, Resend domain and sending key, DNS records, wrangler config files), deploys both workers, seeds credentials, and guides the user through Resend domain verification and admin console first-time setup. The script uses direct API calls to Cloudflare and Resend — no external toolchain dependencies beyond what the project already requires.

## Out of scope

- **Migration of existing deployers** — this feature applies to new installations only; migrating existing deployments to the new setup flow is a separate concern
- **Admin API key rotation or regeneration** — deferred to Admin Console Enhancements
- **Resend API key validation during setup** — the scoped key's validity is inherently proven by successful creation via the Resend API; the full-access key is validated by the provisioning calls themselves
- **Admin email format enforcement beyond basic check** — basic `@` presence check catches obvious typos; full email validation is not in scope
- **Changes to install.sh** — the bootstrap script (prerequisite checks, clone, pnpm install, hand off to setup.sh) remains unchanged; the setup script handles all provisioning
- **Non-interactive mode** — environment variable support for CI/automation is a future enhancement; this feature focuses on the interactive first-run experience
