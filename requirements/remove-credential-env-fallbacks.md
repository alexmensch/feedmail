## Feature: Remove credential env-var fallbacks

The `isAuthorized()` function in `src/api/worker.js` and `getResendApiKey()` in `src/shared/lib/db.js` currently check environment variables first and fall back to the D1 `credentials` table. This creates two competing sources of truth for secrets. This feature removes the env-var path so the D1 credentials table is the single, authoritative source for `ADMIN_API_KEY` and `RESEND_API_KEY`. `DOMAIN` remains an env var by design.

## Out of scope

- Moving `DOMAIN` out of environment variables. `DOMAIN` remains an env var by design since it is configuration, not a secret.
- Adding a UI or admin endpoint for managing credentials in the DB. Credentials are managed via `setup.sh` or direct DB operations.
- Credential rotation or versioning. This feature only changes where credentials are read from, not how they are managed over time.
- Changes to the admin magic-link auth flow itself. This feature is limited to removing the env-var fallback for the two identified credentials.
