## Feature: Auth Session Cleanup

Expired admin sessions and magic link tokens accumulate indefinitely in D1 because nothing deletes them after they expire. This feature adds probabilistic cleanup for both auth tables, triggered on admin requests, and extracts the existing probabilistic cleanup pattern from rate limiting into a shared utility so all three tables use the same mechanism.

## Out of scope

- **Cron-based cleanup** — cleanup runs only on admin requests via probabilistic trigger; no coupling to the feed-check cron job
- **Targeted deletion of individual expired sessions on encounter** — when session validation finds an expired session for the current request, it does not delete that specific row; only the probabilistic sweep cleans up
- **Configurable cleanup probability via admin API** — the 1% probability is a code constant, not a runtime-configurable setting
- **Cleanup of verification_attempts table** — this table is managed by its own existing logic and is not part of this feature
- **Changing rate limit cleanup strategy** — the refactoring (R2) preserves existing behavior exactly; no change to age thresholds, probability, or targeted per-request cleanup
