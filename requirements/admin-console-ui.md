## Feature: Admin Console UI

A server-rendered web interface for managing feedmail, served by the admin Worker established in the Admin Auth features. Exposes admin API functionality through an HTMX-powered UI with CUBE CSS styling. The admin console is a thin client — all data operations go through the existing admin API via `ADMIN_API_KEY`, with the admin Worker acting as an intermediary that renders HTML.

## Out of scope

- **Server-side pagination for subscriber list** — deferred to Admin Console Enhancements feature; subscriber list renders all results without pagination
- **Site configuration editing** (verify limits, rate limits) — deferred to Admin Console Enhancements feature; admin can use the API directly for config changes
- **Subscriber management actions** (edit, delete, change status) — view-only for this iteration; management features will be added later when the admin API supports them
- **Dashboard charts or trend data** — stats are current-point-in-time numbers only; historical trends are a future enhancement
- **Bulk operations** (e.g., delete multiple feeds, export subscribers) — not needed for single-admin use case at this stage
- **Admin console theming or branding customization** — the console has a single, fixed visual design
- **Offline support or service worker caching** — admin console requires network access to function
- **Keyboard shortcuts** — standard browser and form keyboard behavior only
- **Real-time updates** (WebSocket or SSE for live stats) — data refreshes on page load and explicit actions only
