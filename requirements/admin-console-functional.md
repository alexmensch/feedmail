## Feature: Admin Console (Functional)

A plain HTML admin console that exposes all admin API management capabilities through server-rendered forms and pages. The admin Worker acts as an intermediary — it calls the API Worker's admin endpoints using `ADMIN_API_KEY` and renders the results as HTML. All pages use full page reloads with no client-side interactivity beyond browser-native `confirm()` dialogs and existing WebAuthn ceremony JavaScript. Styling is limited to what already exists in the auth pages on master; no new CSS architecture is introduced.

## Out of scope

- **HTMX interactions** — all navigation and form submissions use full page reloads; HTMX is deferred to the admin-console-styled feature
- **CSS architecture or visual design** — no new styling; pages use the minimal CSS already present in the auth page templates on master
- **Responsive layout** — no sidebar layout, no fluid design, no viewport-specific behaviour; deferred to admin-console-styled
- **Dark mode** — deferred to admin-console-styled
- **Server-side pagination for subscriber list** — deferred to admin-console-enhancements; subscriber list renders all results
- **Site configuration editing** (verify limits, rate limits) — deferred to admin-console-enhancements
- **Credential management** (admin email, API keys, Resend key) — deferred to admin-console-enhancements
- **Subscriber management actions** (edit, delete, change status) — view-only for this iteration
- **Dashboard charts or trend data** — stats are current-point-in-time numbers only
- **Bulk operations** (e.g., delete multiple feeds, export subscribers) — not needed at this stage

## Technical Specification

### Summary

This feature replaces the placeholder admin dashboard with a fully functional server-rendered admin console. The Admin Worker acts as an intermediary between the browser and the API Worker — it makes authenticated HTTP requests to the API Worker's admin endpoints using `ADMIN_API_KEY` (read from D1) and renders the JSON responses as HTML pages. All pages use full page reloads with Handlebars templates, reusing the existing styling patterns from the auth pages. The passkey management page is relocated from `/admin/passkeys` to `/admin/settings`. Channel creation is modified to allow creating channels without feeds.

- Confirm dialogs via `onsubmit="return confirm('...')"` (pattern in `admin-passkeys.hbs`)
